AgentOrchestrator 工作原理示意

GitHub 是事实来源；Orchestrator 负责确定性调度；Agent 只提出结构化建议；所有写动作经过策略、状态和幂等校验。

端到端主流程

1. Issue 触发用户添加 agent:autopilot label。
2. Webhook Intake签名校验、payload 限制、delivery 去重。
3. Domain Event归一化为内部事件。
4. State Machinelease + CAS 推进状态。
5. PlanPlanner 输出计划，Plan Reviewer 审核。
6. Implement受控 workspace 产出 diff，实际 changed files 过 policy。
7. PR GatePR review 和 checks 绑定 current head sha。
8. Merge CloseoutMerge API、branch cleanup、final summary、close Issue。

关键保护机制

幂等

  • 每个写动作有 idempotency key。
  • 同 key 同 hash 复用结果。
  • 同 key 不同 hash 进入冲突。

Head SHA 绑定

  • Review、checks、merge-ready 只对当前 head 有效。
  • PR synchronize 会清理旧结论。
  • Merge API 必须传当前 head。

策略阻断

  • deny/high-risk path 自动阻断。
  • pause/no-merge/needs-human 阻止自动推进。
  • Agent 输出不能扩大权限。
