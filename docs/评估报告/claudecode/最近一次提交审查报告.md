## 2.2. 关键发现（按严重度排序）

### 2.2.1. 🔴 严重问题

#### F1. `test/public-api.test.ts:14` —— 测试套件当前在 main 上失败

- **现状**：测试断言列表仍包含已删除的 `'runUiBrowserSmoke'`（line 14），但 `src/index.ts` 已删除该 export。
- **实测验证**：
  ```
  $ node --experimental-strip-types --test test/public-api.test.ts
  ✖ public entrypoint exports only stable operator-facing API
  AssertionError: actual 缺 'runUiBrowserSmoke'
  ```
- **影响面**：CI 的 `check` job（`ci.yml:35` 链式执行 `npm run check` → `npm run test:unit`）每次都红，commit 留下回归。
- **修复方案**：删除 `test/public-api.test.ts:14` 的 `"runUiBrowserSmoke",`。

#### F2. `package-lock.json:15, 502-528` —— lockfile 与 package.json 不同步

- **现状**：`package.json` 已删除 `playwright` devDep，但 `package-lock.json` 根包 `devDependencies` 仍写 `"playwright": "^1.52.0"`（line 15），`node_modules/playwright`（line 502）与 `node_modules/playwright-core`（line 521）整块残留。
- **影响面**：
  - 任意 CI job 跑 `npm ci`（check、coverage、smoke-e2e、smoke-cli）会重新拉取 playwright（~250MB、~30s 额外时间）。
  - 部分 npm 版本下会报 lockfile 不一致错误。
- **修复方案**：执行 `npm install` 重新生成 lockfile 后再提交。

## 2.5. 元评估

- 提交信息中"UI smoke testing belongs in CI as a quality gate"的承诺与 diff 不一致：实际是"完全删除 UI smoke"，CI 中的 `smoke-ui` job 也一并删除，没有替代品。本次删除让仓库失去了**唯一的浏览器级 UI 端到端覆盖**，未来 UI 端 JS 错误（如自动刷新、空态、过滤器）将无任何自动化测试捕获。如果这是有意的取舍，建议在提交信息或 release notes 中明确说明；如果是过渡安排，应在后续 commit 中补一个 HTTP-only 的轻量 smoke job。
- 本次 commit 暴露的真正问题是**"删除不彻底"**：源代码层面删得很干净，但工具脚本（tools/）、依赖契约（package-lock.json）、测试契约（public-api.test.ts）这三类"配套引用"被遗漏。这与 CLAUDE.md 的 SURGICAL CHANGES 原则相违背 —— 应 touch 的就 touch 完整。
